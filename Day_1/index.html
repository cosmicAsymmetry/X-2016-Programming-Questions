Tank Trouble

A general decides to reward all $N$ of his cadets for doing well in their last exam. He places $N$ tanks in a base, and labels them $1$ to $N$. The cadets’ dog tags (again, labeled $1$ to $N$ for the corresponding cadet) are randomly shuffled and distributed among the tanks (one tag per tank).

Then, a game begins. The cadets will wait outside the base and—one by one—each cadet is allowed to go in and open up to $K$ tanks in search of his dog tag. The tags cannot be moved. After the cadet is done, he must close all the tanks and leave the base exactly as it was upon his entry. The cadets are not allowed to communicate with one another during the game.

If all $N$ cadets can find their dog tags, then they win; otherwise, they lose. Before starting the game, the cadets are given a moment to discuss a strategy for maximizing their chances of winning. No cadet has prior knowledge about the distribution of the dog tags.

Assuming optimal play, what is the probability that $N$ cadets will win the game if they are allowed to look at up to $K$ tanks?

Input: Two space-separated integers, $N$ and $K$.

Output: The required probability, correct to $10^{-7}$ precision.

Limit: $2 \leq K \leq N \leq 100000$

Example

Input Output
8 3 0.1467262

Round Pokémon·

Chanaz goes to catch em’ all

One day Chanaz got motivated to be the best, so she planned to catch Pokémon. She has to catch a total of $n$ Pokémon. But sadly, Chanaz is exhausted, so she works as follows :
First she catches $v$ Pokémon, takes rest at the Pokémon Centre , then she catches as many as $[v / k]$ Pokémon, takes rest again, then catches $[v / (k^2)]$ Pokémon and so on, forming the sequence :
$[v], [v / k], [v / k^2], [v / k^3], [v / k^4], [v / k^5], [v / k^6] ...$ The expression $[a / b]$ is regarded as the integral part from dividing number $a$ by number $b$, that is, the Greatest Integer Function(Roof function).

The moment the value of $[v / (k^p)]= 0$, Chanaz stops and goes to sleep. Once she sleeps, she gives up on catching Pokémon. But she wants to have n Pokémon till then.

Chanaz is wondering, what minimum value $v$ can take to let her catch not less than n Pokémon before she falls asleep. Can you help her?

Input: The first two lines contain $x$ and $y$ respectively. The third line contains $x$ numbers giving $z$ for each of the $x$ weeks.

Output: The number of containers used.

Limit: $1 < x \leq 10^9$ and $2 \leq k \leq 10$

Example

Input Output
7 2
59 9		 4
54
Explanation: In the first sample the answer is $v$ = 4. Chanaz catches Pokémon as following: first 4 Pokémon, then 2, then 1, and then Chanaz falls asleep. Thus, she manages to catch 4 + 2 + 1 = 7 Pokémon in the night and complete the task.
In the second sample the answer is $v$ = 54. Chanaz catches Pokémon in the following portions: 54, 6. The total sum is 54 + 6 = 60, that's more than $n$ = 59.

Enigma

Some teachers were inspired by the movie 'The Imitation Game' which was loosely based on the professional and personal history of Alan Turing. A significant portion of the movie revolves around cracking the code generated by Enigma machine which was employed by the German Army. As a joke, these teachers created an Enigma machine of their own to send secret messages to their fellow teachers. As students were the physical messengers of these messages, who lugged around carrying sheets of papers, encryption was the way to go.

For those who are not familiar with the concept of the Enigma machine, please check the Appendix at the end of this question before proceeding with the question.

One of the students decided to crack the code. He snuck into one of the teachers’ room, and found the Enigma machine lying unguarded on the table. It only had 2 rotors with 26 numbers on each rotor (0 to 25). Seizing the opportunity he noted down all the numbers in the order.

1st Rotor: [16, 10, 13, 17, 11, 6, 4, 7, 18, 19, 2, 20, 12, 9, 1, 24, 22, 15, 8, 21, 23, 14, 5, 25, 3, 0]
2nd Rotor: [22, 9, 1, 3, 17, 8, 23, 2, 6, 18, 16, 0, 25, 14, 4, 10, 19, 5, 21, 20, 24, 7, 13, 15, 12, 11]

The principal of the school has a habit of greeting everyone with "good morning everyone", which indicated them to be the principal’s first words in the first message.

The starting point of the rotors for each day was a secret which was known to all the teachers beforehand.

Today's first words were an easy catch as that student was taking the message himself.

Your task is to figure out today's starting point using a C++ program and the given information. Also, write a program to decrypt the messages using the starting point.

Appendix

The enigma machine generates pseudo-random shifts for each character in the message. In this case 2 rotors are used with random numbers (0 to 25) on the rotor disks. Let us take a toy example to understand how the rotor system works

1st Rotor: [0, 1, 2]
2nd Rotor: [0, 1, 2]

Let the starting position be (0,1) and the message be 'abc xyz'. The rotor will operate in the following manner

1st Rotor 2nd Rotor Character Shift Message Encrypted
0 1 0 + 1 = 1 a b
1 1 1 + 1 = 2 b d
2 1 2 + 1 = 3 c f
0 2 0 + 2 = 2 x z
1 2 1 + 2 = 3 y b
2 2 2 + 2 = 4 z d
0 0 0 + 0 = 0    
1 0 1 + 0 = 1    
2 0 2 + 0 = 2    
... ... ... ... ...

and so on
Hence,
Encrypted Message = ‘bdf zbd’

Note

  • No shift is applied on white spaces
  • The message can only consist of 26 alphabets, hence a shift of 1 on 'z' will take you back to the start and yield the encrypted character to be 'a'.

Mastermind AI

We present here a slightly modified version of the popular game of Mastermind. The primary goal of the game is to crack a four digit code set by another player. The code set has to satisfy the following conditions:

  • All the digits should be integers in the interval 0 through 9.
  • No digit should be repeated.

Examples of impermissible codes are, then, 1213, 1123, 1a23 and 2112. Suppose Alice sets the code and Bob has to guess what it is. He gives Alice, as input, a (valid) four digit "trial" code. What Alice outputs, then, is the number of "bulls" and "cows" in the trial code. A cow is defined as a case wherein a digit in the trial code occurs in the actual code but is not in the right place. A bull, on the other hand, means that a digit is not only in the code but also at the correct position. Bob will have ten guesses in which he has to make careful inferences and try to crack the code while minimizing the number of tries. Suppose Alice sets the code as 1234 and Bob's guess is 1546. Then Alice would tell him that his code has one bull and one cow — 1 is its correct position in Alice's code, thus it is a bull, while 4 is a cow as it is in the code but the position guessed is wrong. More examples later.

Now, we’re Alice and you’re Bob.

Round Five ·

Towers of Hanoi

The Tower of Hanoi puzzle was invented by the French mathematician Edouard Lucas in 1883. He was inspired by a legend that tells of a Hindu temple where the puzzle was presented to young priests. At the beginning of time, the priests were given three poles and a stack of 64 gold disks, each disk a little smaller than the one beneath it. Their assignment was to transfer all 64 disks from one of the three poles to another, with two important constraints. They could only move one disk at a time, and they could never place a larger disk on top of a smaller one. The priests worked very efficiently, day and night, moving one disk every second. When they finished their work, the legend said, the temple would crumble into dust and the world would vanish. Although the legend is interesting, you need not worry about the world ending any time soon. The number of moves required to correctly move a tower of 64 disks is, in fact, 18,446,744,073,709,551,615. At a rate of one move per second, that is 584,942,417,355 years! Clearly there is more to this puzzle than meets the eye. We want you to solve the puzzle for n disks. Supposing there are n disks at pole A, print the moves required to move them to pole C with the constraints we mentioned.

Input: The number of disks at pole A at the beginning of the game

Output: Valid moves to move them to pole C in the format given in the example.

Limits: $1\leq n\leq 64$

Example

Input Output
2 1 A to B
2 A to C
3 B to C

Game of Life

In Conway's Game of Life, cells in a grid are used to simulate biological cells. Each cell is considered to be either alive or dead. At each step of the simulation each cell's current status and number of living neighbors is used to determine the status of the cell during the following step of the simulation. In this one-dimensional version, there are $N$ cells numbered $0$ through $N-1$. The number of cells does not change at any point in the simulation. Each cell $i$ is adjacent to cells $i-1$ and $i+1$. Here, the indices are taken modulo $N$ meaning cells $0$ and $N-1$ are also adjacent to eachother. At each step of the simulation, cells with exactly one living neighbor change their status (alive cells become dead, dead cells become alive). For example, if we represent dead cells with a '0' and living cells with a '1', consider the state with 8 cells:

01100101

  • Cells 0 and 6 have two living neighbors.
  • Cells 1, 2, 3, and 4 have one living neighbor.
  • Cells 5 and 7 have no living neighbors.

Thus, at the next step of the simulation, the state would be:
00011101
Given some state of the game, your task is to determine the state immediately preceding it. In some cases there may be more than one answer or no possible answer.

Input: Input will begin with an integer $t$, the number of test cases. Each test case consists of a single line.

Output: For each test case, output the state of the game that precedes the given state. If there is no possible solution, print "No solution". If there are multiple possible solutions, print "Multiple solutions".

Limits: $1\leq t \leq 100$. Each test case can be from 3 to 20 characters long.

Example

Input Output
3
00011101
000001
11110		 01100101
No solution
